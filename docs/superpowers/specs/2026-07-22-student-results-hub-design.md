# 학생 결과 통합 허브 설계

## 배경

교사가 한 학생의 결과를 확인하려면 현재 여러 화면을 오가야 한다.

- 시험별 제출과 답안·필기는 `/teacher/exam/[id]`와 `/teacher/attempt/[attemptId]`에 있다.
- 학생 프로필과 성장 리포트는 `/teacher/users`에 있다.
- 시험별·학생별 분석은 `/teacher/dashboard`의 탭에 있다.

기능이 존재해도 진입 경로와 명칭이 분산되어 있어 발견하기 어렵고, 같은 학생과 시험 맥락을 반복해서 찾아야 한다. 이 설계는 기존 제출 상세 경로를 학생 결과 통합 허브로 확장해 답안, 필기, 리포트, 분석을 한 흐름에서 제공한다.

## 목표

- 시험 상세의 학생 결과 표를 학생별 결과 확인의 명확한 출발점으로 만든다.
- 한 학생의 현재 시험 답안, 필기, 리포트, 분석을 한 화면에서 전환한다.
- 원시험과 여러 재시험 회차를 같은 학생 맥락에서 비교한다.
- 기존 사용자 관리와 분석 대시보드의 관련 링크를 통합 허브로 연결한다.
- 사용할 수 없는 유료 기능도 숨기지 않고 가치와 제한 이유를 이해할 수 있게 한다.
- 기존 경로, 저장 데이터, 분석 계산식, 플랜 정책을 유지한다.

## 비목표

- 분석 계산식이나 추천 알고리즘 변경
- 플랜별 권한 정책 변경
- 데이터베이스 스키마 또는 제출 식별자 마이그레이션
- 교사용 학생 관리와 전체 시험 분석 화면 제거
- 새로운 학생 메시지나 피드백 기능 추가

## 검토한 접근

### 1. 기존 제출 상세 경로를 통합 허브로 확장 — 채택

`/teacher/attempt/[attemptId]`를 유지하고 상단 학생·시험·회차 문맥과 네 개의 결과 탭을 추가한다. 기존 링크가 깨지지 않고, 현재 제출 중심 데이터 모델과 가장 잘 맞는다.

### 2. 시험·학생 전용 새 경로 추가

`/teacher/exam/[examId]/student/[studentId]`와 같은 경로는 의미가 명확하지만 학생 ID가 없는 과거·익명 제출을 별도로 처리해야 한다. 기존 제출 상세와 역할도 중복된다.

### 3. 시험 상세 내부 패널

학생 행에서 빠르게 열 수 있지만 PDF 필기와 상세 답안에 필요한 넓은 화면을 확보하기 어렵고 모바일 경험이 불리하다.

## 정보 구조와 이동 흐름

### 시험 상세

- 학생 결과 표의 작업 버튼은 필기 유무와 관계없이 `학생 결과 보기`로 통일한다.
- 각 행에는 `필기 저장됨`, `원시험`, `재시험`, 회차 수와 같은 상태를 보조 정보로 표시한다.
- 버튼은 해당 제출의 `/teacher/attempt/[attemptId]`로 이동한다.
- 시험 상세 상단의 전체 시험 분석 링크는 기존 시험 분석 화면으로 계속 연결한다.

### 학생 결과 통합 허브

- 상단 경로는 `시험 상세 > 학생 이름`으로 표시한다.
- 학생 이름, 시험 제목, 점수, 제출 시각, 원시험·재시험 상태를 한 영역에 요약한다.
- 원시험과 재시험 회차 전환기를 제공한다. 재시험에는 원시험 대비 점수 증감을 표시한다.
- 본문에는 `답안`, `필기`, `리포트`, `분석` 탭을 고정한다.
- 기본 탭은 `답안`이다.
- 탭은 `?view=answers`, `?view=handwriting`, `?view=report`, `?view=analytics`로 직접 연결할 수 있다.
- 회차를 바꾸면 선택한 제출 경로로 이동하면서 현재 `view` 값을 유지한다.
- 뒤로 가기 동선은 현재 시험의 학생 결과 표를 향한다.

### 다른 화면의 진입점

- 사용자 관리의 필기 열람 링크는 통합 허브의 `필기` 탭으로 연결한다.
- 사용자 관리의 학생 성장 리포트 링크는 대표 제출을 기준으로 `리포트` 탭에 연결한다.
- 학생 분석 화면에서 특정 학생·제출을 여는 링크는 `분석` 탭으로 연결한다.
- 기존에 저장되거나 공유된 `/teacher/attempt/[attemptId]` 링크는 기본 `답안` 탭으로 정상 동작한다.

## 탭별 역할

### 답안

학생이 무엇을 제출했는지 확인하는 원자료 화면이다.

- 현재 점수와 정답·오답·미응답·미채점 요약
- 전체, 오답, 미응답 필터
- 문항별 선택 답, 정답, 채점 상태, 배점
- 서술형 하위 문항 답변과 검토 상태
- 학생 질문과 교사 답변

### 필기

학생이 어떻게 풀었는지 확인하고 교사가 피드백하는 과정 화면이다.

- 기존 PDF 필기 뷰어
- 학생 필기, 교사 마크업, 합성 보기 전환
- 문항별 필기 요약과 해당 위치 이동
- 교사 피드백 임시 저장과 학생에게 돌려주기
- 필기 데이터 내보내기와 기존 플랜별 PDF 내보내기

### 리포트

현재 시험 결과를 빠르게 읽거나 출력하는 요약 화면이다.

- 점수, 정답·오답·미응답 요약
- 주요 오답과 취약 유형
- 원시험 대비 선택한 재시험의 변화
- 교사 피드백 요약
- 인쇄 친화적인 단일 흐름
- 누적 성장 요약과 PDF 내보내기는 기존 플랜 정책에 따라 부분 잠금

### 분석

원인을 진단하고 다음 행동을 결정하는 대화형 화면이다.

- 현재 시험의 취약 유형과 반복 실수
- 느린 문항, 재방문 문항, 답 변경 문항
- 이탈 기록과 집중도 단서
- 원시험·재시험 비교
- 추천 재시험 문항과 다음 학습 행동
- 동일 학생 연결이 가능한 경우 누적 점수 추이와 반복 약점을 보조 영역으로 제공

## 구성 요소 경계

- `StudentResultHeader`: 시험·학생·점수·제출 상태와 시험 상세 복귀 동선을 소유한다.
- `AttemptSwitcher`: 동일 시험에서 같은 학생의 원시험·재시험 회차를 정렬하고 선택한다.
- `StudentResultTabs`: URL과 선택 탭을 동기화하고 접근 가능한 탭 내비게이션을 제공한다.
- `AnswersPanel`: 답안 요약, 필터, 문항 결과, 학생 질문을 소유한다.
- `HandwritingPanel`: PDF, 필기, 마크업, 필기 관련 오류 상태를 소유한다.
- `ReportPanel`: 현재 시험 요약, 비교 요약, 인쇄 레이아웃을 소유한다.
- `AnalyticsPanel`: 현재 시험 진단과 누적 학생 분석을 소유한다.
- `LockedFeaturePanel`: 잠긴 기능의 가치, 제한 이유, 제공 플랜과 업그레이드 동선을 일관되게 표시한다.

기존 제출 상세의 계산과 상호작용은 위 단위로 옮기되 결과 계산 함수와 저장 게이트웨이는 재사용한다. 하나의 큰 페이지 컴포넌트가 네 탭의 세부 구현을 모두 소유하지 않게 한다.

## 데이터 흐름

### 최초 진입

1. 경로의 `attemptId`로 현재 제출을 불러온다.
2. 제출의 `examId`로 시험을 불러온다.
3. 현재 제출과 시험으로 헤더, 회차 전환기, 답안 탭을 렌더링한다.
4. `view`가 없거나 유효하지 않으면 답안 탭을 선택한다.

### 동일 학생과 회차 연결

동일 학생 판단은 안정성이 높은 식별자부터 적용한다.

1. `studentProfileId`
2. `studentId`
3. 프로젝트의 기존 학생 프로필 호환 규칙

이 규칙으로 현재 시험의 원시험과 재시험을 모아 시간순·관계순으로 정렬한다. 안정적으로 연결할 수 없는 과거 제출은 현재 제출만 보여주고, 누적 영역에 연결 제한 이유를 표시한다. 이름만 같은 학생을 임의로 합치지 않는다.

### 지연 로딩

- 최초 진입에서는 현재 제출, 시험, 회차 목록, 답안에 필요한 데이터만 준비한다.
- PDF 원본과 무거운 필기 데이터는 필기 탭을 처음 열 때 불러온다.
- 전체 시험 이력과 누적 분석은 리포트 또는 분석 탭을 처음 열 때 불러온다.
- 이미 성공적으로 불러온 탭 데이터는 현재 화면 생명주기 동안 재사용한다.

## 플랜 제한과 빈 상태

- 네 탭은 플랜과 데이터 유무에 관계없이 항상 노출한다.
- 권한이 없으면 기능을 숨기지 않고 미리보기, 제한 이유, 제공 플랜, 업그레이드 동선을 표시한다.
- 필기 권한이 있지만 해당 제출에 필기가 없으면 잠금 화면이 아니라 `저장된 필기가 없습니다` 빈 상태를 표시한다.
- 현재 시험 리포트의 기본 요약은 제공한다. 누적 성장 영역과 PDF 내보내기는 기존 권한 정책에 따라 부분 잠금한다.
- 분석 가능한 문항 데이터가 없으면 저장 점수와 제출 정보는 유지하고 계산 불가능한 항목만 빈 상태로 표시한다.

## 오류 처리

- 제출을 찾지 못하면 기존 찾을 수 없음 화면을 표시한다.
- 시험을 찾지 못하면 제출 당시 저장된 시험 제목과 점수를 사용하고 답안·분석 제한을 안내한다.
- 필기 파일 로딩이 실패하면 필기 탭 안에서 원인을 안내하고 재시도 동선을 제공한다. 다른 탭은 계속 사용할 수 있다.
- 누적 제출 로딩이 실패하면 현재 시험 결과는 유지하고 누적 영역만 재시도할 수 있게 한다.
- 유효하지 않은 `view` 값은 답안 탭으로 복구한다.
- 접근 권한 검사는 기존 교사 세션과 조직 범위 검사를 유지한다.

## 반응형과 접근성

- 데스크톱에서는 학생 결과 요약과 회차 전환기를 같은 상단 영역에 배치한다.
- 모바일에서는 상단 정보를 세로로 쌓고 회차 전환기를 선택 메뉴로 바꾼다.
- 네 탭은 모바일에서 2열 2행으로 배치해 잘림과 가로 스크롤을 피한다.
- 답안은 문항별 행을 유지하되 작은 화면에서 핵심 값이 먼저 보이도록 열 우선순위를 조정한다.
- 탭은 적절한 `tablist`, `tab`, `tabpanel` 역할과 `aria-selected`, 연결 ID를 사용한다.
- 키보드 방향키와 Tab 키로 탭과 주요 작업을 이동할 수 있게 한다.
- 잠금, 빈 상태, 오류 상태를 색상만으로 구분하지 않는다.
- 포커스 표시와 최소 터치 영역을 유지한다.

## 테스트 전략

### 단위 테스트

- `view` 값 파싱과 기본값 복구
- 원시험·재시험 관계와 회차 정렬
- 원시험 대비 점수 변화량
- 동일 학생 연결 우선순위와 이름 기반 오결합 방지
- 플랜 권한, 필기 데이터 유무에 따른 잠금·빈 상태 결정

### 통합 및 E2E 테스트

- 시험 상세의 `학생 결과 보기`로 통합 허브 진입
- 네 탭 전환과 URL 동기화
- 직접 링크, 새로고침, 브라우저 뒤로 가기
- 회차 전환 시 현재 탭 유지와 점수 변화 표시
- 사용자 관리의 필기·리포트 링크가 해당 탭으로 이동
- 권한 없음, 필기 없음, 필기 로딩 실패, 시험 정보 일부 누락 상태
- 기존 PDF 필기 열람, 교사 마크업, 피드백 저장·반환, 인쇄 기능 회귀
- 모바일 탭과 회차 선택기, 답안 목록 레이아웃
- 탭 키보드 탐색과 접근성 속성

## 완료 기준

- 교사가 시험 상세에서 한 번의 명확한 작업으로 학생 결과 통합 허브에 진입한다.
- 답안, 필기, 리포트, 분석을 다른 상위 화면으로 돌아가지 않고 전환할 수 있다.
- 원시험과 여러 재시험을 전환하면서 선택 탭과 학생 문맥이 유지된다.
- 사용자 관리와 분석 화면의 관련 링크가 통합 허브의 정확한 탭으로 연결된다.
- 잠김, 데이터 없음, 부분 로딩 실패가 서로 다른 상태로 명확하게 표현된다.
- 기존 제출 상세 링크와 핵심 필기·피드백 기능이 회귀 없이 동작한다.
- 데스크톱과 모바일에서 주요 작업이 잘리거나 숨지 않는다.
